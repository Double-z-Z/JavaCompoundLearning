## 核心原则

**边学边沉淀，拒绝事后整理！**

**复利核心**：每个知识点必须建立至少2个双向链接，形成知识网络。

## 工作流程（保持不变）

### 1. 识别知识类型

在学习过程中，实时识别内容类型：

| 内容类型 | 判断标准 | 保存位置 | 使用模板 |
|---------|---------|---------|---------|
| **练习代码** | 实现的功能、对比测试 | `03-Practice/drills/` | 练习记录模板 |
| **核心概念** | 原理、机制、设计思想 | `02-Knowledge/<主题>/concepts/` | 原子笔记模板 |
| **思维误区** | 错误理解→正确理解的转变 | `03-Practice/reflections/` | 反思记录模板 |
| **对话感悟** | 学习心得、思维升级 | `03-Practice/reflections/` | 反思记录模板 |
| **项目实战** | 多阶段完整项目 | `01-Projects/` | 项目笔记模板（由项目拆解Skill创建） |

### 2. 即时保存策略（保持不变）

#### **场景 A：完成练习**

```
用户完成代码实现
    ↓
判断是否属于某个项目？
    ├── 是 → 保存到：03-Practice/drills/<日期>-<项目名>-<阶段>.md
    │         在练习记录中链接到：[[项目笔记#Phase X]]
    │         更新项目笔记中的"关联练习"链接
    │
    └── 否 → 保存到：03-Practice/drills/<日期>-<主题>.md
    ↓
使用模板：练习记录模板.md
包含：
- 实现了什么
- 对比测试结果
- 关联的概念（更新相关概念的 mastery）
    ↓
继续下一个知识点
```

#### **场景 B：讨论核心概念**

```
用户提问原理/机制
    ↓
AI 解释 + 横向拓展
    ↓
立即保存到：02-Knowledge/<主题>/concepts/<概念>.md
使用模板：原子笔记模板.md
包含：
- 一句话定义
- 核心理解（自由写作）
- 关键关联（至少2个双向链接 + 关联原因）
- 我的误区
- mastery=20 (初识)
    ↓
继续深入或切换到下一个话题
```

#### **场景 C：纠正思维误区**

```
用户表达错误理解
    ↓
AI 追问 + 引导反思
    ↓
用户顿悟
    ↓
立即记录到：03-Practice/reflections/<日期>-<主题>-dialogue.md
使用模板：反思记录模板.md
包含：
- 问题 → 我的回答 → AI 追问 → 最终理解
- 顿悟时刻
- 相关概念的 mastery 更新（+20分深度理解）
    ↓
继续对话
```

### 3. 掌握度（mastery）更新流程

每个原子笔记的 frontmatter 中包含 `mastery` 字段，由AI根据学习活动自动更新。

#### 分值等级

| 分值 | 等级 | 判定标准 |
|-----|------|---------|
| 0-30 | 🌱 初识 | 刚创建笔记，理解表面 |
| 31-60 | 🌿 理解 | 能解释概念，完成相关练习 |
| 61-85 | 🍎 应用 | 能在项目中正确使用 |
| 86-100 | 🌳 掌握 | 能向他人讲解，处理边界情况 |

#### 更新触发条件

**+10-20分：完成练习**
- 完成相关 drill，验证理解
- 在练习中正确应用概念
- 在练习记录的AI评价中记录：`- [[概念A]]: +15分 (正确应用)`

**+10-15分：建立新关联**
- 发现与已有知识的新连接
- 更新笔记添加双向链接
- 在原子笔记的AI评价中记录更新

**+15-25分：教学/讲解**
- 向他人解释这个概念
- 在对话中纠正他人误区
- 在反思记录的AI评价中记录更新

**+5-10分：复习巩固**
- 间隔复习后仍能准确回忆
- 完成自测问题

**-5-10分：发现理解偏差**
- 练习中犯错
- 对话中发现误解
- 创建错误档案时记录

#### 更新操作

在原子笔记的 `🤖 AI评价` 区域记录：
```markdown
### 掌握度评估
- 当前等级：🌿理解
- 更新记录：
  - 2026-04-13: mastery=20 (初建笔记)
  - 2026-04-14: mastery=45 (完成练习，正确应用)
```

同时更新 frontmatter：
```yaml
---
mastery: 45
---
```

## 5. 对话结束时的汇总（保持不变）

### **必做检查点**


## 6. 质量红线（保持不变）

### ❌ 禁止

- 先写大笔记，再事后拆分
- 只记录到临时文件，不结构化
- 对话结束才想起来保存
- 原子笔记少于2个双向链接

### ✅ 必须

- 每个知识点讨论完立即保存
- 按模板结构化内容
- 添加知识关联（至少 2 个 + 关联原因）
- 更新 .agent/profile.md
- 更新相关概念的 mastery 字段

---

## 7. 触发条件（保持不变）

**自动触发**：
- 用户说"整理笔记"
- 完成一个练习任务
- 讨论完一个核心概念
- 纠正一个思维误区

**手动触发**：
- 用户明确要求保存到某位置
- 对话自然段落结束

## 8. 示例流程（保持不变，展示 mastery 更新）

### 示例：学习 LongAdder

```
1. 用户实现 LongAdderCounter
   → 立即保存到 03-Practice/drills/2026-04-11-concurrent-counters.md
   → AI评价：[[LongAdder]] mastery +15分 (正确应用)

2. 用户提问"LongAdder 的 base 和 cell 作用"
   → AI 解释
   → 立即保存到 02-Knowledge/concurrency/concepts/LongAdder.md
   → 建立关联：[[分段锁思想]]、[[缓存行伪共享]]
   → mastery=20 (初识)

3. 用户说"我以为 Cell 直接 value++"
   → AI 追问，引导发现误区
   → 立即记录到 03-Practice/reflections/2026-04-11-longadder-dialogue.md
   → 顿悟：理解了CAS仍然必要
   → AI评价：[[LongAdder]] mastery +20分 (深度理解，纠正误区)
   → 更新 LongAdder.md: mastery=40

4. 对话结束
   → 创建汇总：03-Practice/reflections/2026-04-11-summary.md
   → 更新 .agent/profile.md 的已掌握知识点
   → 记录：LongAdder (mastery=40, 🌿理解)
```
