# 评估触发

> **触发条件**
> - 用户声明完成项目/练习
> - 完成架构设计讨论
> - 完成故障排查
> - 达成里程碑目标

---

## 评估流程（"练习即评估"闭环）

```
用户完成项目/练习
       │
       ▼
┌─────────────────┐
│ AI检测完成状态   │
│ （代码提交/声明）│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 引导填写评估卡片 │
│ （基于模板）     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ AI给出评估建议   │
│ （等级+置信度）  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 生成评估文件     │
│ 更新current.json │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 更新MOC雷达图   │
└─────────────────┘
```

|------|---------|---------|

## AI评估职责

### 1. 检查新评估证据

| 证据类型 | 映射维度 |
|---------|---------|
| 新原子笔记 | 核心技术知识 |
| 新项目代码 | 工程实践 |
| 新错误档案 | 问题分析 |
| 新架构讨论 | 架构设计 |

### 2. 置信度判定

- **高置信度**：代码+对话+错误档案交叉验证
- **中置信度**：仅有笔记或单次对话
- **低置信度**：仅有计划或自我报告

### 3. 更新评估档案

- 更新 `.agent/assessment/current.json`
- 更新 `04-Maps/MOC-能力雷达图.md`
- 更新 `.agent/profile.md` 中的 mastery 字段

### 4. 推荐学习路径

- 识别最低分维度
- 推荐下一学习主题
- 生成带评估验证点的学习计划

## 里程碑定义

由学习者自主设定，非固定时间：
- 完成一个主题的系统学习（如"完成NIO专题"）
- 达到某个能力等级（如"并发编程达到L3"）
- 完成一个综合项目（如"完成高并发计数器服务"）
- 积累一定量的知识资产（如"产出10个原子笔记"）
