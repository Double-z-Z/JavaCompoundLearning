# 复利工程工作流程

## Skill 触发条件与核心流程

以下场景出现时，**主动按对应 skill 执行**：

### 整理笔记
**触发**：讨论完核心概念 / 完成练习 / 纠正思维误区 / 用户说"整理笔记"。
**核心规则**：
- 练习代码 → `03-Practice/drills/<日期>-<主题>.md`
- 核心概念 → `02-Knowledge/<主题>/concepts/<概念>.md`（使用原子笔记模板，至少 2 个双向链接 + 关联原因）
- 思维误区/对话感悟 → `03-Practice/reflections/<日期>-<主题>-dialogue.md`
- **边学边沉淀，拒绝事后整理！**

### 项目拆解
**触发**：用户想做一个新项目作为学习载体。
**核心规则**：
- 读取 `.agent/profile.md` 分析已掌握知识
- 分析项目涉及的知识点、与已掌握知识的关联
- 生成循序渐进的学习路径（Phase 1/2/3）
- **自动创建** `01-Projects/<项目名>.md`（基于项目笔记模板）
- 预测易错环节，引用历史错误模式

### 评估触发
**触发**：用户声明完成项目/练习 / 完成架构设计 / 完成故障排查 / 达成里程碑。
**核心规则**：
- 引导填写评估卡片（基于模板）
- 给出评估建议（等级 + 置信度）
- 更新 `.agent/assessment/current.json`、`.agent/profile.md`、`04-Maps/`

### 学习推荐
**触发**：用户说"开始今日学习" / 每次新对话开场 / 完成一个主题后。
**核心规则**：
- 分析 `.agent/profile.md`、`02-Knowledge/`、`03-Practice/mistakes/`、`01-Projects/` 数据
- 生成学习状态分析 + 3 个选项（继续项目 / 复习巩固 / 探索新知）
- 引用 mastery 分数、MOC 进度、历史错误预警
- 用户选择后，**记录到 `.agent/profile.md`** 的"当前学习计划"

### 强化练习
**触发**：错误归档后 / 用户说"强化练习"。
**核心规则**：基于最近错误，生成 3 道变式练习（相似场景不同异常、多知识点结合、真实边界情况）。

### 整理图谱
**触发**：对话结束时（如有知识产出）。
**核心规则**：更新 `04-Maps/` 下的 MOC 文件和知识图谱，确保新知识点有至少 2 个链接。

## Mastery 更新规则

每个原子笔记 frontmatter 包含 `mastery` 字段，根据学习活动自动更新：

| 分值 | 等级 | 标准 |
|-----|------|---------|
| 0-30 | 🌱 初识 | 刚创建笔记，理解表面 |
| 31-60 | 🌿 理解 | 能解释概念，完成相关练习 |
| 61-85 | 🍎 应用 | 能在项目中正确使用 |
| 86-100 | 🌳 掌握 | 能向他人讲解，处理边界情况 |

**更新幅度参考**：
- 完成练习：+10~20
- 建立新关联：+10~15
- 教学/纠正误区：+15~25
- 复习巩固：+5~10
- 发现理解偏差：-5~-10

**必须在原子笔记和 `.agent/profile.md` 中同步记录 mastery 变化。**

## 质量红线

### ❌ 禁止
- 直接给出完整代码答案（除非用户明确要求）
- 先写大笔记，再事后拆分
- 只记录到临时文件，不结构化
- 对话结束才想起来保存
- 原子笔记少于 2 个双向链接
- 一次性给出完整技术文档

### ✅ 必须
- 每个知识点讨论完立即保存
- 按模板结构化内容
- 添加知识关联（至少 2 个 + 关联原因）
- 更新 `.agent/profile.md`
- 更新相关概念的 `mastery` 字段
- 长回复加 TL;DR

---

## 对话后必做

完成主要回应后，**必须**：

1. **询问是否需要归档本次对话要点**
2. **建议更新哪些知识库文件**（具体到路径和文件名）
3. **基于当前进度预测并建议下一步学习路径**
4. **检查是否触发评估** — 若涉及学习成果，主动触发评估流程
5. **如有新知识点产出，更新相关 MOC 和知识图谱**
