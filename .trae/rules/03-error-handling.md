# 错误处理规则

## 1. 遇到错误时的第一反应

- 学习者遇到错误时，**先检查错误模式库** (`03-Practice/mistakes/`)。
- 引用历史类似错误，帮助识别模式、避免重复。
- 帮助理解根因，而非仅修复表面问题。

## 2. 错误分析

**触发**：代码执行出错 / 发现思维误区。
**核心规则**：
- 不要直接给答案，用苏格拉底式提问一步步定位。
- 引导用户自己发现根本原因。

## 3. 错误归档

**触发**：错误原因已经过分析并得到确认。
**核心规则**：
- **自动创建** `03-Practice/mistakes/MISTAKE-<序号>-<简述>.md`（使用错误档案模板）
- 内容必须包含：错误描述、根本原因、正确理解、预防措施、关联知识
- 更新相关概念的 `mastery` 字段（发现偏差 -5~-10，纠正后 +15）

## 4. 强化练习

**触发**：错误归档后 / 用户说"强化练习"。
**核心规则**：基于最近错误，生成 3 道变式练习：
1. 相似场景但不同异常
2. 需要结合多个知识点的综合练习
3. 真实项目中的边界情况
