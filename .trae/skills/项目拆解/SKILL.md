# 项目拆解

> **路径配置**（从 `.agent/config.md` 读取）
> - 学习者画像：`.agent/profile.md`
> - 错误档案：`03-Practice/mistakes/`
> - 学习路线图：`LEARNING-ROADMAP.md`
> - 项目笔记：`01-Projects/`
> - 项目笔记模板：`05-Templates/项目笔记模板.md`

---

## 工作流程

### 1. 分析阶段

请根据 `.agent/profile.md` 中的学习者水平：

1. **分析这个项目涉及哪些核心知识点**
2. **这些知识点与已掌握的有什么关联**
   - 参考 `.agent/profile.md` 中的"已掌握知识点"
3. **生成一个循序渐进的学习路径，每个阶段有可验证的代码产出**
4. **预测可能在哪些环节犯错**
   - 参考 `03-Practice/mistakes/` 中的历史错误模式

### 2. 生成项目笔记（自动）

根据分析结果，**自动创建项目笔记**：

**文件路径**: `01-Projects/<项目名称>.md`

**创建内容**（基于模板填充）：
- 项目目标
- 涉及知识点表格（含角色定位和练习链接占位符）
- 架构设计框架（结构 + 决策 + 交互流程）
- Phase 1/2/3 框架（目标 + 验证方式 + 练习占位符）
- 性能测试记录表
- 项目特有的坑记录区
- 跨概念综合洞察区

### 3. 生成分阶段练习计划

为每个 Phase 预设练习记录占位符：
- `03-Practice/drills/<日期>-<项目名>-phase1.md`
- `03-Practice/drills/<日期>-<项目名>-phase2.md`
- `03-Practice/drills/<日期>-<项目名>-phase3.md`

## 使用流程

```
用户：我想做一个 RPC 框架项目
    ↓
AI：读取 .agent/profile.md 分析已掌握知识
    ↓
AI：分析 RPC 框架涉及的知识点
    ↓
AI：生成 3 个 Phase 的学习路径
    ↓
AI：自动创建 01-Projects/rpc-framework.md
    ↓
AI：提示用户开始 Phase 1，并创建对应练习记录
```

---

## 项目笔记 vs 练习记录的分工

| 内容 | 记录位置 | 说明 |
|-----|---------|------|
| 项目架构设计 | 项目笔记 | 跨组件的整体设计 |
| 单个技术点验证 | 练习记录 | 链接到项目笔记的对应 Phase |
| 性能对比数据 | 项目笔记 | 综合性的 benchmark |
| 代码实现过程 | 练习记录 | 具体代码和思考 |
| 跨概念洞察 | 项目笔记 | 整合多个知识点的理解 |
| 遇到的错误 | 错误档案 | 项目笔记链接到错误档案 |

## 质量检查

创建项目笔记后检查：
- [ ] 涉及知识点是否都已链接到原子笔记？
- [ ] 每个 Phase 是否有明确的可验证产出？
- [ ] 是否引用了历史错误模式作为预警？
- [ ] 性能测试方案是否可行？
