# 项目：并发编程能力测试

> 测试你对Java内存模型和多线程编程的掌握程度

## 测试项目1：高并发计数器系统

### 需求描述
实现一个线程安全的计数器，支持以下功能：
- 支持多个线程同时递增计数器
- 支持获取当前计数值
- 支持重置计数器
- 性能要求：10个线程各递增100万次，总时间 < 1秒

### 知识点考察
- volatile的可见性
- 原子类（AtomicLong）的使用
- synchronized与Lock的区别
- CAS原理与ABA问题

### 进阶挑战
- 实现一个分段计数器（类似ConcurrentHashMap的分段锁思想）
- 对比不同实现方式的性能差异

---

## 测试项目2：生产者消费者队列

### 需求描述
实现一个多生产者多消费者的任务调度系统：
- 使用BlockingQueue作为任务队列
- 3个生产者线程，每秒生产10个任务
- 5个消费者线程，处理任务（模拟耗时100ms）
- 任务队列大小限制为100

### 知识点考察
- BlockingQueue的put/take操作
- 线程池的合理配置
- 拒绝策略的选择
- 优雅关闭线程池

### 进阶挑战
- 实现优先级任务队列
- 监控队列积压情况并动态调整消费者数量

---

## 测试项目3：简易线程池实现

### 需求描述
手写一个简化版线程池，支持：
- 核心线程数和最大线程数配置
- 任务队列（有界队列）
- 拒绝策略（抛出异常/丢弃/调用者运行）
- 线程空闲回收机制

### 知识点考察
- 线程池核心参数（corePoolSize、maxPoolSize、keepAliveTime）
- 任务提交与执行流程
- Worker线程的生命周期管理
- 线程安全的状态管理

### 进阶挑战
- 支持定时任务调度
- 支持线程池状态监控（活跃线程数、队列大小等）

---

## 测试通过标准

1. **功能正确性**：所有测试用例通过，无并发问题
2. **性能达标**：满足各项目的性能要求
3. **代码质量**：良好的注释、异常处理、资源释放
4. **知识讲解**：能清晰解释实现原理和设计选择

## 学习产出
