# 拒绝策略

## 一句话定义
当线程池达到最大容量（线程数=max + 队列满）时，决定**如何处理新任务**的策略机制。


## 核心机制
| 策略 | 行为 | 适用场景 |
|------|------|----------|
| AbortPolicy | 抛出RejectedExecutionException | 关键任务，需要明确失败 |
| CallerRunsPolicy | 调用者线程执行 | 低负载，需要保证任务执行 |
| DiscardPolicy | 静默丢弃 | 非关键任务，允许丢失 |
| DiscardOldestPolicy | 丢弃最旧的任务 | 时间敏感任务，优先执行新任务 |


## 为什么重要
- 系统过载时的**自我保护机制**
- 防止任务无限堆积导致OOM
- 不同业务场景需要不同的降级策略


## 与已学知识的关联
- [[线程池]] → 拒绝策略是线程池的最后防线
- 熔断器模式 → 类似的系统保护思想
- 背压机制 → 流量控制的另一种实现


## 类比理解
🔄 这就像：**餐厅等位系统**
- AbortPolicy = 告诉顾客"今天不接待了"
- CallerRunsPolicy = 让推荐人来帮忙服务
- DiscardPolicy = 默默取消预约（顾客不知道）
- DiscardOldestPolicy = 取消最早的预约，让新顾客进来


## 自定义拒绝策略示例
```java
new RejectedExecutionHandler() {
    @Override
    public void rejectedExecution(Runnable r, ThreadPoolExecutor executor) {
        // 记录日志
        log.warn("Task rejected: {}", r);
        // 降级处理：保存到数据库，稍后重试
        saveToDb(r);
    }
};
```
