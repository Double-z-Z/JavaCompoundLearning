# 线程池

## 一句话定义
线程池是**管理和复用线程资源**的机制，通过池化技术减少线程创建销毁开销，并提供任务调度、流量控制功能。


## 核心机制
1. **线程复用**：工作线程循环获取任务，避免频繁创建销毁
2. **分层执行**：核心线程→任务队列→非核心线程→拒绝策略
3. **状态管理**：RUNNING→SHUTDOWN→STOP→TIDYING→TERMINATED


## 核心参数
| 参数 | 含义 | 设计要点 |
|------|------|----------|
| corePoolSize | 核心线程数 | 保持活跃的最小线程数 |
| maximumPoolSize | 最大线程数 | 可扩展的最大线程数 |
| workQueue | 工作队列 | 任务缓冲，决定排队策略 |
| handler | 拒绝策略 | 队列满时的处理策略 |


## 为什么重要
- 线程创建开销大（系统调用+内存分配）
- 防止线程数量失控导致系统崩溃
- 提供统一的任务管理和监控能力


## 与已学知识的关联
- 阻塞队列 → 线程池的任务缓冲机制
- [[拒绝策略]] → 流量控制的关键组件
- 线程状态 → 理解工作线程的生命周期
- [[futex]] → 阻塞队列底层可能使用park/unpark


## 常见误区
- ❌ "线程池越大越好" → 过多线程导致上下文切换开销
- ❌ "队列越大越好" → 无界队列可能导致内存溢出
- ❌ "直接shutdownNow()" → 可能中断正在执行的任务


## 代码示例
```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    4,                      // 核心线程数
    8,                      // 最大线程数
    60L, TimeUnit.SECONDS,  // 空闲线程存活时间
    new LinkedBlockingQueue<>(100),  // 有界队列
    new ThreadFactoryBuilder().setNameFormat("pool-%d").build(),
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
);
```
