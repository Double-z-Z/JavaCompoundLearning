# 缓存行伪共享

## 一句话定义
多个CPU核心频繁修改**同一缓存行内的不同变量**，导致缓存行在核心间反复失效，引发性能下降的现象。


## 核心机制
1. **缓存行**：CPU缓存的最小单位（通常64字节）
2. **MESI协议**：多核缓存一致性协议
3. **伪共享**：不同变量在同一缓存行，修改时互相影响


## 为什么重要
- 可使多线程程序性能下降10倍以上
- LongAdder的@Contended注解专门解决此问题
- 高性能并发编程必须考虑硬件层面


## 与已学知识的关联
- [[LongAdder]] → Cell类使用@Contended避免伪共享
- CPU缓存 → 理解缓存行是理解伪共享的前提
- MESI协议 → 缓存一致性的底层机制


## 类比理解
🔄 这就像：**图书馆的座位预约**
- 伪共享 = 一个预约单包含多个座位，一个人改自己的座位，其他人的预约单也要重新打印
- 避免伪共享 = 每个座位单独一张预约单，互不影响


## 解决方案
```java
@sun.misc.Contended  // JDK8+，自动填充到64字节
static final class Cell {
    volatile long value;
}
```
