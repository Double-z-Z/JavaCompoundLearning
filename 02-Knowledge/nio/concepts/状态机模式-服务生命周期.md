# 状态机模式 - 服务生命周期管理

## 一句话定义
使用有限状态机（FSM）管理服务从启动到停止的完整生命周期，确保状态流转清晰、行为可预测。


## 核心理解

### 为什么需要状态机？
服务生命周期涉及多个阶段（NEW → RUNNING → STOPPING → STOPPED），每个阶段允许的行为不同。使用布尔标志（如 `isRunning`）无法表达复杂状态，容易导致：
- 状态不一致（多个标志组合出非法状态）
- 重复关闭、关闭顺序错误等问题
- 代码可读性差

### 状态机设计要点

```
NEW → STARTING → RUNNING → STOPPING → STOPPED
```

| 状态 | 允许的操作 | 禁止的操作 |
|------|-----------|-----------|
| NEW | start() | stop(), 处理请求 |
| RUNNING | 处理请求, stop() | start() |
| STOPPING | 完成正在处理的请求 | 接受新请求 |
| STOPPED | 无 | 所有操作 |

### 与布尔标志的对比

**❌ 布尔标志方式（问题）**
```java
private volatile boolean isRunning;
private volatile boolean isShuttingDown;

// 问题：可以出现 isRunning=false, isShuttingDown=false 的非法状态
// 问题：循环条件难以表达"正在关闭但队列未空"
while (isRunning || (isShuttingDown && !queue.isEmpty())) // 复杂难懂
```

**✅ 状态机方式（清晰）**
```java
public enum State { NEW, RUNNING, SHUTTING_DOWN, STOPPED }

// 清晰表达循环条件
while (state == State.RUNNING || 
       (state == State.SHUTTING_DOWN && !queue.isEmpty()))
```


## 关键关联

- [[NIO-Selector-模式]] - 关联原因：Selector 的注册/取消也需要状态管理，与生命周期状态协同
- [[线程池-ExecutorService]] - 关联原因：Java 标准线程池使用状态机管理生命周期（RUNNING → SHUTDOWN → STOP → TERMINATED）
- [[优雅关闭-Graceful-Shutdown]] - 关联原因：状态机是实现优雅关闭的基础，SHUTTING_DOWN 状态确保消息处理完毕才退出


## 我的误区与疑问

- ❌ **误区**：之前用 `isRunning` 布尔标志，发现无法表达"正在关闭但还需处理剩余消息"的状态
- ✅ **纠正**：引入 `SHUTTING_DOWN` 中间状态，明确区分"不再接受新消息"和"完全停止"


## 代码与实践

### ServerHandler 状态机实现
```java
public enum State {
    NEW,           // 新建状态
    RUNNING,       // 运行中（处理 I/O 和消息）
    SHUTTING_DOWN, // 正在关闭（不再接受新消息，处理队列剩余消息）
    STOPPED        // 已停止
}

private volatile State state = State.NEW;

public void run() {
    state = State.RUNNING;
    try {
        // 关键：RUNNING 或 (SHUTTING_DOWN 且队列未空) 时继续循环
        while (state == State.RUNNING || 
               (state == State.SHUTTING_DOWN && !messageQueue.isEmpty())) {
            // 处理 I/O 和消息...
        }
    } finally {
        state = State.STOPPED;
        cleanup();
    }
}

public void shutdown() {
    if (state != State.RUNNING) return;
    state = State.SHUTTING_DOWN;  // 进入关闭状态
    enqueueShutdownNotice();       // 放入关闭通知
    selector.wakeup();             // 唤醒处理
}
```
