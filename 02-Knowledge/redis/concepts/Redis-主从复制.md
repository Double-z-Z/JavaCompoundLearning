# Redis 主从复制

## 一句话定义
主从复制是 Redis 实现数据冗余和读写分离的基础机制：主节点（Master）处理写操作，数据异步复制到一个或多个从节点（Slave），从节点处理读操作。


## 核心理解

### 复制流程：全量同步 + 增量同步

```
┌─────────┐                    ┌─────────┐
│  Slave  │ ─── REPLICAOF ───► │  Master │
│ (从节点) │                    │ (主节点) │
└────┬────┘                    └────┬────┘
     │                              │
     │◄────────── RDB 文件 ─────────┤  1. 生成 RDB（fork + BGSAVE）
     │                              │
     │◄──────── 缓冲区命令 ─────────┤  2. 发送 RDB 生成期间的写命令
     │                              │
     │◄──────── 实时复制流 ─────────┤  3. 持续同步新命令
     │                              │
```

**三个阶段**：

| 阶段 | 动作 | 关键技术 |
|------|------|---------|
| **全量同步** | 传输 RDB 文件 | `fork() + BGSAVE`（COW）|
| **延迟补偿** | 发送复制缓冲区命令 | 复制积压缓冲区（replication backlog）|
| **增量同步** | 持续命令传播 | 主从保持长连接，异步发送 |

### 复制缓冲区（Replication Backlog）

```
Master 内部维护：
┌─────────────────────────────────────┐
│      复制积压缓冲区（固定大小）        │
│  ┌─────────────────────────────┐    │
│  │ cmd1 │ cmd2 │ cmd3 │ ...   │    │
│  │  ◄──────────────────────►   │    │
│  │      offset 递增            │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘

Slave 断开重连：
├── 上报自己的 offset
├── Master 检查 offset 是否在缓冲区内
├── 是 → 增量同步（发送缺失命令）
└── 否 → 全量同步（RDB 太大，重新来）
```

**默认大小**：1MB（可配置 `repl-backlog-size`）

### 主从复制的权衡

```
一致性 vs 性能：
┌─────────────────────────────────────────┐
│  Master 写成功 ──► 立即返回客户端        │
│       │                                │
│       ▼ 异步复制                        │
│  Slave 接收 ──► 可能有延迟（毫秒级）      │
└─────────────────────────────────────────┘

结果：主从数据最终一致，非强一致
```


## 关键关联

- [[Redis-持久化]] - 关联原因：全量同步使用 BGSAVE 生成 RDB，依赖 fork+COW 机制
- [[Redis-哨兵模式]] - 关联原因：哨兵监控主从节点，自动故障转移
- [[NIO-Selector]] - 关联原因：主从复制的网络层使用异步非阻塞传输，类似 NIO 思想
- [[NIO-广播队列]] - 关联原因：Redis 7.0 共享复制缓冲区的设计与广播队列的多消费者共享模式相似


## 我的误区与疑问

- ❌ 误区：以为主从复制是强一致的（实际是最终一致）
- ❓ 疑问：复制缓冲区满了会怎样？（最慢的 Slave 决定清理进度，可能触发全量同步）


## 代码与实践

```bash
# 配置从节点
REPLICAOF 192.168.1.1 6379

# 查看复制状态
INFO replication

# 配置复制缓冲区大小
CONFIG SET repl-backlog-size 64mb

# 配置最小从节点数
CONFIG SET min-slaves-to-write 1
CONFIG SET min-slaves-max-lag 10
```
