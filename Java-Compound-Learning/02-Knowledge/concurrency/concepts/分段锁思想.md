# 分段锁思想

## 一句话定义
将**一个共享资源拆分为多个独立分段**，不同线程操作不同分段，从而降低竞争、提升并发性能的设计思想。


## 核心机制
1. **资源分段**：将大锁拆分为多个小锁（或CAS变量）
2. **哈希映射**：通过哈希算法将线程映射到不同分段
3. **局部处理**：每个线程只操作自己的分段，互不干扰


## 为什么重要
- 将串行操作变为并行操作
- 冲突概率从O(n)降低到O(n/m)，m为分段数
- 是高并发系统的核心设计模式之一


## 与已学知识的关联
- [[LongAdder]] → Cell数组就是分段思想的实现
- [[ConcurrentHashMap]] → JDK7的Segment数组，JDK8的桶链表分段
- [[线程池]] → 多个工作线程分别处理不同任务


## 类比理解
🔄 这就像：**银行柜台**
- 不分段 = 1个窗口，所有人排一队
- 分段 = 多个窗口，客户分散办理，吞吐量提升


## 常见误区
- ❌ "分段越多越好" → 分段过多会增加内存开销和汇总成本
- ❌ "分段后完全无竞争" → 哈希冲突时仍可能竞争同一分段


## 代码示例
```java
// 分段思想的核心：哈希映射
int index = threadId & (segmentCount - 1);  // 位运算取模
segments[index].increment();  // 操作对应分段
```
