# 用户态与内核态切换

## 一句话定义
CPU从执行用户程序切换到执行操作系统内核代码的过程，涉及**特权级切换**和**上下文保存**。


## 核心机制
1. **触发方式**：系统调用、中断、异常
2. **切换开销**：保存/恢复寄存器、切换页表、刷新TLB
3. **耗时**：约100纳秒（仅切换本身）


## 为什么重要
- 系统调用是用户程序访问硬件的唯一途径
- 频繁的态切换是性能瓶颈（futex的设计目标就是减少切换）
- 理解"为什么用户态CAS比synchronized快"


## 与已学知识的关联
- [[futex]] → 减少态切换是futex的核心优化点
- [[线程状态]] → 线程阻塞必须进入内核态执行schedule()
- [[CAS]] → 用户态CAS避免了态切换


## 性能对比
| 操作 | 耗时 |
|-----|------|
| 用户态CAS | ~20纳秒 |
| 态切换（系统调用） | ~100纳秒 |
| 上下文切换 | ~1-10微秒 |


## 常见误区
- ❌ "态切换就是上下文切换" → 态切换是特权级改变，上下文切换是换线程执行
- ❌ "系统调用都很慢" → 现代CPU有快速系统调用指令（sysenter/syscall）
