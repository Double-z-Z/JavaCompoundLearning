## 产品功能

### 这是什么？

Java复利学习工程是一个**个人知识管理系统**，通过结合AI智能体和Obsidian笔记工具，帮助Java学习者：

- ✅ **建立知识网络** - 每个新概念自动链接到已有知识
- ✅ **避免重复犯错** - 错误模式归档，AI主动预警
- ✅ **智能学习推荐** - 基于掌握程度和遗忘曲线推荐学习内容
- ✅ **项目驱动学习** - 通过实战项目整合多个知识点
- ✅ **复利式增长** - 知识资产持续积累，越学越快

### 核心能力

| 能力          | 说明                         |
| ----------- | -------------------------- |
| **AI学习合伙人** | 基于你的学习历史，推荐最适合的学习路径        |
| **错误模式库**   | 记录、分类、预警，避免重复踩坑            |
| **掌握度追踪**   | 0-100分 mastery 系统，精准定位薄弱环节 |
| **知识图谱**    | MOC主题地图，可视化知识网络            |
| **项目实战**    | 多阶段项目，从理论到应用               |

### 适用人群

- 📚 有一定基础，希望**系统化提升**的Java开发者
- 🎯 追求**深度理解**而非表面知识的学习者
- 🔄 相信**持续积累**产生复利效应的长期主义者
- 🤖 愿意与**AI协作**，提升学习效率的先行者

## 核心理念

### 1. 复利学习思维

> "学习就像投资，今天的知识要成为明天学习的基础"

**传统学习**：学一个忘一个，每次都从零开始  
**复利学习**：每个新知识链接到已有知识，形成网络，产生复利

```
传统学习：A → B → C → D （线性，易遗忘）
复利学习：A ↔ B ↔ C ↔ D （网络，互相强化）
```

### 2. Obsidian + AI 知识库

**Obsidian提供**：
- 双向链接 - 知识网络的可视化
- Dataview - 动态查询掌握度
- MOC - 主题地图导航
- 本地优先 - 数据完全自主可控

**AI提供**：
- 智能推荐 - 基于数据而非主观
- 知识连接 - 主动发现关联
- 错误预警 - 历史模式识别
- 苏格拉底式引导 - 深度理解而非记忆

### 3. 计划 vs 推荐

- **人制定计划** - 决定学什么、花多少时间
- **AI负责推荐** - 基于掌握度、遗忘曲线、知识网络给出建议
- **人做决策** - 选择接受或调整推荐
- **AI记录执行** - 追踪实际完成情况，优化后续推荐

## 运作机制

### 三阶段工作流

```
┌─────────────────────────────────────────────────────────────┐
│                     AI合伙人工作流                           │
├─────────────────┬─────────────────────┬─────────────────────┤
│   阶段一：准备   │    阶段二：对话      │    阶段三：沉淀      │
├─────────────────┼─────────────────────┼─────────────────────┤
│ 读取学习者画像   │ 知识连接与引导       │ 询问归档需求         │
│ 扫描知识库状态   │ 错误模式识别         │ 建议更新知识库       │
│ 生成学习推荐     │ 代码引导（不给答案）  │ 预测下一步学习       │
│                 │ 长内容加TL;DR       │ 记录学习决策         │
└─────────────────┴─────────────────────┴─────────────────────┘
```

### Skill 协作体系

```
用户说"开始今日学习"
    ↓
【学习推荐】Skill
    ├── 分析：profile.md + 知识库扫描
    ├── 输出：学习状态 + 3个推荐选项 + 预警
    └── 记录：用户决策 → profile.md
    ↓
用户选择学习内容
    ↓
【苏格拉底式引导】/【横向拓展】Skill（按需）
    ↓
【整理笔记】Skill（实时）
    ├── 保存到 02-Knowledge/（原子笔记）
    ├── 保存到 03-Practice/（练习/反思）
    └── 更新 mastery 分数
    ↓
遇到错误
    ↓
【错误分析】→ 【错误归档】→ 【强化练习】Skill
    ↓
【项目拆解】Skill（项目规划时）
    └── 创建 01-Projects/（项目笔记）
```

### 数据流转

```
.agent/profile.md（中央记录）
    ├── 已掌握知识点 + mastery分数
    ├── 当前学习计划
    ├── 历史决策记录
    └── 学习偏好

        ↑↓ 双向同步

各模块数据
    ├── 02-Knowledge/    - 结构化知识库
    ├── 03-Practice/     - 练习、错误、反思
    ├── 04-Maps/         - MOC主题地图
    └── 01-Projects/     - 项目实战
```

## 快速开始

### 环境准备

1. **安装 Obsidian** - [下载地址](https://obsidian.md/)
2. **安装必要插件**：
   - Dataview（数据查询）
   - Graph View（知识图谱可视化）
3. **配置 Trae IDE** 或其他支持AI Agent的编辑器

### 项目初始化

```bash
# 克隆项目
git clone <repository-url>
cd Java-Compound-Learning

# 用 Obsidian 打开项目文件夹
# 在 Obsidian 中配置 Dataview 插件
```

### 配置 AI 合伙人

1. **复制智能体提示词**：
   - 文件位置：`.trae/prompts/`
   - 将内容复制到 Trae 的智能体设置中

2. **配置学习者画像**：
   - 编辑 `.agent/profile.md`
   - 填写你的学习背景、已掌握知识点

3. **开始第一次对话**：
   - 对 AI 说："开始今日学习"
   - AI 会分析你的状态并给出推荐

### 目录结构说明

```
Java-Compound-Learning/
├── .agent/                    # AI配置
│   ├── config.md             # 路径配置
│   └── profile.md            # 学习者画像（需配置）
├── 01-Projects/               # 项目实战
├── 02-Knowledge/              # 知识库
│   └── <主题>/
│       └── concepts/         # 原子笔记
├── 03-Practice/               # 练习与反思
│   ├── drills/               # 练习记录
│   ├── mistakes/             # 错误档案
│   └── reflections/          # 对话反思
├── 04-Maps/                   # 知识图谱（MOC）
├── 05-Templates/              # 笔记模板
└── .trae/skills/              # AI Skill定义
```

## 参与贡献

### 面向开发者

如果你想了解项目实现或贡献代码：

#### 技术栈

- **Obsidian** - 知识库管理
- **Markdown** - 内容格式
- **Dataview** - 动态查询
- **AI Agent** - Trae IDE 或其他支持Skill的系统

#### Skill 开发

项目使用 **Skill 体系** 定义 AI 行为：

```yaml
# Skill 结构
.trai/skills/<skill-name>/
└── SKILL.md          # Skill定义

# Skill 格式
---
name: Skill名称
description: 触发时机和条件
---

# Skill 逻辑（Markdown格式）
```

**核心 Skill**：
- `学习推荐` - 分析状态，生成推荐
- `项目拆解` - 创建项目笔记和学习路径
- `整理笔记` - 实时沉淀知识
- `错误分析` - 引导定位问题
- `错误归档` - 归档错误模式
- `强化练习` - 生成变式练习

#### 贡献方式

1. **提交 Issue** - 报告问题或建议
2. **提交 PR** - 改进 Skill 或添加新功能
3. **分享案例** - 分享你的学习实践
