### 核心数据结构

#### 1. profile.md（学习者画像）

```yaml
---
learner_profile:
  level: intermediate
  
mastered_concepts:
  - name: 线程池
    mastery: 75              # 0-100
    last_reviewed: 2026-04-10
    
current_plan:
  active_project: NIO聊天室
  
learning_history:
  - date: 2026-04-13
    choice: 继续项目
---
```

**作用**：AI 读取此文件了解学习者状态，生成个性化推荐。

#### 2. 原子笔记（概念存储）

```markdown
---
created: 2026-04-12
tags: [concurrency]
status: 🌿理解
mastery: 60
related_concepts:
  - [[CAS]]
  - [[分段锁思想]]
---

# LongAdder

## 一句话定义
...

## 核心理解
...
```

**作用**：存储单个概念，通过 `related_concepts` 建立知识网络。

#### 3. 错误档案

```markdown
---
created: 2026-04-11
tags: [mistake, concurrency]
error_id: MISTAKE-001
status: active
---

# 错误标题

## 错误描述
...

## 正确理解
...

## 预防措施
...
```

**作用**：归档错误模式，AI 扫描此目录进行预警。

## Skill 系统

### Skill 是什么？

Skill = AI 的"操作手册"，定义在特定场景下 AI 如何响应。

### Skill 文件结构

```markdown
---
name: Skill名称
description: 触发时机和条件
---

# Skill 逻辑

## 工作流程
1. 步骤1
2. 步骤2

## 输入/输出
### 输入
- 输入项

### 输出
- 输出项

## 示例
示例代码...
```

### 核心 Skill

| Skill    | 触发条件        | 核心逻辑                                 |
| -------- | ----------- | ------------------------------------ |
| **学习推荐** | 用户说"开始今日学习" | 读取 profile → 扫描知识库 → 计算推荐度 → 输出选项    |
| **整理笔记** | 对话结束        | 识别内容类型 → 选择模板 → 保存到对应目录 → 更新 mastery |
| **错误分析** | 用户遇到错误      | 扫描错误档案 → 查找类似错误 → 苏格拉底式提问 → 引导定位根因   |
| **错误归档** | 错误原因确认      | 创建错误档案 → 关联知识点 → 生成强化练习              |
| **项目拆解** | 开始新项目       | 分析知识点 → 生成学习路径 → 创建项目笔记              |

### Skill 调用链

```
用户：开始今日学习
    ↓
【学习推荐】Skill
    ├── 读取 .agent/profile.md
    ├── 扫描 02-Knowledge/（计算 mastery）
    ├── 扫描 03-Practice/mistakes/（识别活跃错误）
    ├── 计算推荐度（项目连续性 + 遗忘风险 + 关联度）
    └── 输出推荐选项
    ↓
用户选择
    ↓
学习过程中
    ↓
【整理笔记】Skill（实时）
    ├── 识别内容类型（概念/练习/反思）
    ├── 选择模板（05-Templates/）
    ├── 填充内容 + 建立链接
    └── 保存到对应目录
    ↓
遇到错误
    ↓
【错误分析】→ 【错误归档】→ 【强化练习】
    ↓
对话结束
    ↓
【整理笔记】Skill（沉淀）
    └── 更新 profile.md
```

## 推荐算法

### 推荐度计算

```python
推荐度 = (
    项目连续性 * 0.3 +      # 有进行中的项目优先
    遗忘风险 * 0.25 +        # 接近遗忘临界点优先复习
    知识关联度 * 0.2 +       # 与最近学习关联度高优先
    前置依赖 * 0.15 +        # 项目前置知识优先
    网络密度 * 0.1           # 知识孤岛优先连接
)
```

### 遗忘曲线计算

```python
# 基于艾宾浩斯遗忘曲线
遗忘风险 = f(mastery, days_since_last_review)

# 示例
mastery = 55
days = 7
遗忘风险 = 0.8  # 高风险，建议复习

mastery = 80
days = 7
遗忘风险 = 0.2  # 低风险，可以暂缓
```

## 数据流转

### 对话生命周期

```
1. Session 开始
   ├── 读取 .agent/config.md（路径配置）
   ├── 读取 .agent/profile.md（学习者画像）
   └── 加载到 AI 上下文

2. 用户输入
   └── AI 解析意图

3. Skill 调用
   ├── 识别场景（学习/错误/项目等）
   ├── 调用对应 Skill
   └── 执行 Skill 逻辑

4. 文件操作
   ├── 读取知识库（查询）
   └── 写入知识库（更新）

5. 响应生成
   └── 基于数据和模板生成回复

6. Session 结束
   └── 更新 profile.md（记录决策）
```

### 关键文件操作

| 操作 | 读取 | 写入 |
|-----|------|------|
| 开场推荐 | profile.md, 知识库, 错误档案 | - |
| 学习概念 | 相关原子笔记 | 新原子笔记 |
| 完成练习 | 项目笔记 | 练习记录, 更新 mastery |
| 遇到错误 | 错误档案 | 错误档案（更新） |
| 对话结束 | - | profile.md（记录） |

## 扩展机制

### 如何添加新 Skill

1. 创建目录：`.trae/skills/<skill-name>/` 和 `.agents/skills/<skill-name>/`
2. 编写 `SKILL.md`：定义触发条件和逻辑（两个目录内容保持一致）
3. 测试验证
4. 更新文档

### 如何自定义模板

1. 复制 `05-Templates/` 中的模板
2. 修改字段和格式
3. 更新 Skill 引用

### 如何集成其他工具

**Obsidian 插件**：
- Dataview：动态查询掌握度
- Graph View：可视化知识网络

**AI 工具**：
- 任何支持自定义提示词的工具
- 需要适配 Skill 调用机制
