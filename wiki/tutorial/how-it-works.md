### 场景 1：学习新概念

**你说**："什么是 LongAdder？"

**AI 响应**：

```
这让我想起你之前学的 AtomicLong...

AtomicLong 使用单个变量 + CAS 实现原子递增。
LongAdder 也是原子递增，但用了"分段"思想：

┌─────────────────────────────────────┐
│  AtomicLong（单个变量）              │
│  所有线程竞争同一个变量               │
│  高并发时 CAS 冲突严重                │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  LongAdder（多个 Cell）              │
│  线程分散到不同 Cell 操作             │
│  减少冲突，提高并发性能               │
└─────────────────────────────────────┘

就像银行柜台：
- AtomicLong = 1 个窗口，所有人排一队
- LongAdder = 多个窗口，客户分散办理
```

**你**："明白了"

**AI**："建议保存到知识库？"

**你**："好"

**AI 自动完成**：
- 创建原子笔记
- 建立与 AtomicLong、CAS 的链接
- 设置掌握度 20 分

### 场景 2：遇到错误

**你说**："代码报错了"

**AI 响应**：

```
🤔 让我们分析一下：
1. shutdown() 后线程池处于什么状态？
2. 此时提交任务会发生什么？
3. 正确的关闭流程是什么？
```

**你**：（思考后回答）

**AI**：引导你找到根因

**你**："找到原因了，是关闭后又提交了"

**AI**：
```
✅ 正确！

shutdown() 后线程池状态变为 SHUTDOWN，
不再接收新任务，此时提交会抛异常。

建议：
1. 提交任务前检查 isShutdown()
2. 或使用 awaitTermination() 等待完成

需要归档这个错误吗？
```

## 交互原则

### AI 不会做的

❌ **不给完整代码** - 除非明确要求  
❌ **不代替你思考** - 用苏格拉底式提问引导  
❌ **不强制计划** - 推荐仅供参考，你决定  

### AI 会做的

✅ **用你已知的解释新知识** - "这让我想起你之前学的..."  
✅ **主动发现知识关联** - 建立双向链接  
✅ **引用历史错误** - 避免重复踩坑  
✅ **在遗忘临界点提醒复习** - 基于艾宾浩斯曲线  
✅ **自动整理归档** - 你只需确认

## 常用指令

### 开始对话
- "开始今日学习" - AI 分析状态并推荐
- "我想学习 XXX" - 直接开始特定主题
- "继续 XXX 项目" - 继续进行中的项目

### 学习过程中
- "举个例子" - 切换到横向拓展模式（类比解释）
- "深入理解" - 切换到苏格拉底式引导
- "我遇到错误了" - 启动错误分析流程
- "保存到知识库" - 手动触发归档

### 结束对话
- "今天先到这里" - 进入沉淀阶段
- "整理笔记" - 立即归档当前内容

## 使用体验

### 你会感受到

🎯 **个性化** - AI 越来越了解你，推荐越来越精准  
🔗 **连接感** - 新知识自然连接到已有知识  
📈 **成长感** - 看得见掌握度提升，网络密度增加  
🛡️ **安全感** - 错误被记录，不会再犯  

### 你需要做的

1. **说"开始今日学习"** - 让 AI 了解你的状态
2. **做出选择** - 决定学什么，保持主动权
3. **主动思考** - 回答 AI 的引导问题
4. **确认归档** - 将学习成果固化

### AI 会帮你做的

1. **分析状态** - 读取你的画像和知识库
2. **生成推荐** - 基于数据给出建议
3. **建立连接** - 发现新旧知识的关联
4. **整理归档** - 自动保存到正确位置
5. **规划复习** - 在遗忘临界点提醒你
